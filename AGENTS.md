# Repository Agent Notes

- Work in an isolated worktree and branch for each change. Do not rewrite, delete, or force-clean another agent's dirty worktree.
- Every PR body must begin with `1.) agent {number}` and `2.) purpose: {one-line purpose}` — the `pr-scope` CI job hard-fails without them. It re-runs only on push or close/reopen, not on a body edit, so fix the body first and then retrigger.
- Before declaring a PR merge-ready, merge its branch into *current* main locally and build/test the overlapping crates. Green per-PR CI runs against a stale merge base; a trait or schema change landed on main since branching breaks the combined tree without any PR check noticing.
- Use `gh` for GitHub issue, PR, review, and merge state. Prefer one tight review pass and only a small follow-up loop when there is a concrete blocker.
- After any wait, force-push, PR body edit, CI rerun, or concurrent merge, re-check PR state, head SHA, base SHA, check rollup, and linked issue state with `gh` before reviewing or merging. Stale checks and closed/superseded PRs are not merge evidence.
- Run Cargo verification sequentially per target directory, or give concurrent commands separate `CARGO_TARGET_DIR` values. Missing rlibs, object files, or temp dirs from a shared fresh target are local harness races until reproduced with a clean sequential run.
- When a review or incident exposes a persistent, non-overfit lesson, preserve it as general guidance. Read-only reviewers should report the candidate guidance in the review body; coordinators or follow-up authors should land accepted guidance here and in `.claude/skills/review-pr/SKILL.md` from a separate worktree/PR.
- Keep durable guidance phrased as reusable invariants or failure modes. Avoid issue numbers, branch names, one-off file lines, and examples that will go stale unless they are explicitly called out as examples.
- Security/privacy docs cannot rely on one top-level disclaimer to narrow later absolute claims. Wording such as "all", "every", "owns", "guarantees", or "resistance" needs local shipped-vs-roadmap scope where it appears.
- "Zero history", "zero trace", and "stealth" claims must name the durable sinks covered by tests: journals, cassettes, telemetry, logs, engine profiles, browser cache/storage, crash reports, and OS temp files. If OS, browser, network, or remote-service traces remain outside control, say that locally where the claim appears.
- Encryption claims must name the protected data, key source, integrity mode, rotation/recovery story, and fail-closed behavior. "Encrypted" is not a security property until the threat model and unauthenticated/corrupt-data handling are pinned.
- Browser stealth must mean local privacy, profile isolation, retention suppression, and policy enforcement. Do not merge CAPTCHA solving, anti-bot bypass, or fingerprint-spoofing escalation under "stealth" without an explicit safety/legal review and a separate product decision.
- Runtime-visible contract changes must keep the public descriptions in sync. When routes, status codes, response fields, schemas, or agent/SDK surfaces change, update OpenAPI and generated-client-facing docs in the same slice.
- Cassette and replay-fork import defaults must use authenticated durable retention. Plaintext cassettes are compatibility/test fixtures only; any helper that reads or writes them must be explicitly named unsafe so untrusted imports cannot silently replay attacker-chosen responses.
- Taint-to-beatbox is a runtime dispatch invariant, not a claim established by the bridge crate alone. Do not claim live prompt-injection exfiltration resistance until an agent-facing `tempod`/headless path routes page-tainted non-browser compute through `tempo-toolexec` and a live beatbox canary proves `net:Deny`, `secrets:[]`, and no egress.
- Raw accessibility trees and `tempo-observe` compiler inputs stay in engine adapters or fixtures. `tempod`-side runtime crates should expose only `tempo-schema` `CompiledObservation`/`ObservationDiff` values; native Servo engine-side compilation remains a deferred lane gate until wired and tested.
- Public enum-valued fields must advertise the exact runtime wire names. Prefer deriving/reusing the same schema helper as the serde type; otherwise add an OpenAPI/contract test that pins every value the runtime can emit.
- Model-facing tool schemas must be self-contained and match the runtime parser. Do not publish opaque object placeholders, unresolved `$ref`s, or multiple aliases for the same argument when the caller needs one canonical shape.
- Clients that wrap structured operation envelopes as `Result` must parse the authoritative payload status before mutating local state. HTTP 2xx only means the envelope was delivered; `step_error`, unknown, missing, or non-applied statuses are errors unless the contract says otherwise.
- Adding a field to a public Rust schema struct is also a source-compat change. `serde(default)` protects old JSON payloads, but every workspace struct literal still needs a scan/update and downstream compile check.
- Compact wire-format changes must preserve compatibility in both directions. When default, empty, or omitted fields change, tests should prove compact serialization, populated serialization, and default-filled deserialization so older and newer SDKs keep agreeing.
- Borrowed serializers, budget probes, and other wire-shape proxies must mirror every `serde` default/skip rule on the public type. Boundary-sized payload tests should fail if the proxy counts bytes differently from the real serialized payload.
- Do not commit realistic secret, token, password, or credential literals, even in tests. Build scanner-safe fixtures from clearly inert fragments while still proving redaction and non-leak behavior.
- Secret-bearing HTTP clients must validate configured base URLs before building requests. Production keys should go only to pinned secure origins; loopback or insecure fixtures need an explicitly named opt-in so tests do not normalize unsafe live configuration.
- Network policy decisions must bind to the concrete endpoint that will actually be used after DNS, proxy resolution, redirects, and retries. Validating only a URL string, hostname, or first resolved address leaves room for rebinding or alternate-socket bypasses.
- Raw crawl dispatch is scheduler-only. SDK-facing network execution should prefer connection-pinned capabilities that expose an already-open stream plus socket-safe request parts, not the raw URL. Any fallback checked-dispatch client must connect only to the policy-checked socket; handing the original URL back to a client that can re-resolve it reopens SSRF/TOCTOU.
- Loopback, Host, and Origin checks are boundary and CSRF defenses, not authentication. Any control plane that can drive sessions, tools, or browser state needs an unguessable same-user capability even when bound to `127.0.0.1`.
- Unauthenticated browser-control serving helpers must be explicitly named as test-only or unsafe. A production-looking default, builder, or fixture that silently disables auth is a security regression even if it binds to loopback.
- Engine/control IPC must fail closed on platforms without an implemented peer-auth transport. Do not add a permissive fallback because a path is "local"; wire same-user peer credentials, named-pipe ACLs, app-private sockets, or an explicit unsupported gate first.
- A security bridge with live unit/canary coverage is not shipped product wiring. If a crate like `tempo-toolexec` is not called from the agent/headless/MCP/BiDi runtime path, docs must say the composition is deferred, and CI should pin that status until a production-path sentinel lands.
- Offloading blocking work to OS threads or blocking pools is not a resource cap. Add shared in-flight limits around thread-spawning paths that can be triggered by protocol clients.
- Client connection caps are not live-work caps when handlers spawn detached blocking work. Any client-triggered `spawn_blocking` path that can drive engine work or side effects needs an independent in-flight quota or a real cancellation signal that reaches the engine boundary.
- Page-derived data is not limited to compiled observations. Extraction, script-evaluation, and other DOM/page read results must carry provenance in the return type and across IPC/wire envelopes before they can reach an agent or policy decision.
- Operational metadata that exposes dependency state, capacity, policy, or topology is control-plane data. Guard it with the same auth/host/origin boundary unless the route is intentionally public and boring, like a static liveness check.
- Stateful protocol surfaces need live-state quotas in addition to per-frame or per-body caps; repeated valid commands can be a resource attack even when each request is small.
- A size cap checked only after fully materializing remote-driven JSON, DOM, screenshot, log, or tool-result data is not a memory bound. Enforce the cap while reading, collecting, diffing, or serializing unless the producer is already independently bounded.
- Bounded serialization helpers must measure the source `Serialize` value before materializing an owned `Value`; converting to an owned tree before the cap check is still post-allocation rejection.
- Model/tool result envelopes should carry one authoritative structured payload. Text fallbacks must be summaries, not a second serialized copy of large JSON or binary data.
- Untrusted remote tool catalogs must not weaken local side-effect policy. If a remote descriptor lacks trusted side-effect metadata, classify the action at the strongest supported side effect before applying threshold origin rules.
- OpenAPI discovery is not execution authority. Any direct OpenAPI request path must floor untrusted descriptor operations to at least `Send`, preserve stronger local inference, and reject model-supplied Authorization, cookie, API-key, token, secret, credential, and password-like parameters unless a server-side explicit secret binding supplies them.
- UI-local state transitions and teardown/cancel paths must stay bounded independently of backend health. Moving I/O off-thread is incomplete if local controls or shutdown still serialize behind blocking transport work.
- Async UI/worker reconciliation must preserve ownership boundaries. Never replay a cloned domain object from a stale UI snapshot into a worker or over a completed worker result when the remote/backend side can mutate fields inside that object; send narrow local deltas instead.
- Security gates must exercise the production call path, not just a helper with the intended behavior. If model-facing or network-facing serialization is certified, assert the live request builder uses that exact serializer.
- Page-controlled metadata belongs inside the same provenance framing as visible page text. Escaped-but-bare attributes in model prompts are still unlabeled untrusted input.
- More code is not more optimized. Prefer the smallest change that proves the invariant, removes duplicated paths, or makes an existing contract honest.
- Cross-platform work must stay Android/mobile friendly by default: no desktop-only IPC, filesystem, RAM, process, or windowing assumptions above the engine/transport boundary. Tempo follows the code-backed Servo platform matrix for every Servo-available target, with platform-specific adapters kept thin.
- `tokio::time::timeout` around a resource-creating call is not cancellation: the browser/remote side may still create the resource after the future is dropped. Every timeout arm on a create-style call needs a reap or track path, or retry loops grow the system without bound.
- When bounding one await below an upstream deadline, bound the whole path: setup and cleanup awaits added by the fix (enable, close, teardown) inherit the transport's default timeout and can still blow the deadline being defended. Timeout values are product behavior — derive them from the caller's bound minus recovery margin, not from what quiets a test.
- Per-resource policy interception must exist before the resource's first navigation or use. Create at an inert state (e.g. `about:blank`), install the policy hooks, then drive the real work through the already-policed resource; and keep block attribution on the resource that caused it.
- A child process the daemon auto-starts must not outlive the daemon on any exit path. Signal death runs no drops, so daemon-side cleanup is insufficient; the child needs its own parent-death watch (reparenting is the signal), or every daemon restart leaks the child and everything it spawned.
- When a daemon owns both a child-process supervisor and an IPC-reconnect monitor, write down the coordination story: who restarts, who re-attaches, and what happens after either gives up. A permanent return in one loop must not strand the other.
- When adding a filter to the observation pipeline (visibility, layout, policy), enumerate the carve-outs earlier stages made for downstream detectors and decide explicitly whether the filter applies to them; a uniform prune silently narrows every prior carve-out. Geometry filters must distinguish "not rendered" from "degenerate": an element occupying layout space along one axis (an empty container a later action fills) is still a legitimate action/extract target.
- Any retention or classification rule enforced in injected page JS must be mirrored in the host-side parser as the trust backstop, and each side needs its own reverted-fix-sensitive test; a string-containment assertion on the script text pins nothing behaviorally.
- Curated security-sentinel test lists are append-only. A guard that asserts a denial test exists and passes pins against deletion and rename, but not against an assertion flip paired with an inverted production change — diffs touching trust boundaries or `*_denies_*`/`*_blocks_*` tests still need human review.
- Removing or shortening a wait, grace window, or recheck on a trust-boundary path needs a justification that names the exact window losing coverage and what still enforces the invariant inside that window. "Already covered elsewhere" must cite the covering mechanism, not just assert it.
- When a caller starts trusting callee-embedded data it previously re-derived (a returned diff, status, or measurement), the callee's contract — trait docs, schema comments — must be updated in the same slice to require what the caller now assumes, or a future implementation silently weakens the guarantee.
- Dropping per-write fsync from a durable file needs an integrity story, not just a recency argument. WAL-style posture requires WAL-style recovery: either checksummed records or detection-plus-self-heal for interior corruption (a garbage line that is well-terminated), not only torn-tail repair. Heal-by-truncation must be gated so a misconfigured key/policy on a healthy store can never trigger it.
- Never revalidate a cached verdict about a shared mutable file (a quarantined region, a known-good prefix, a parsed snapshot) by length or mtime equality when another writer can heal or re-record in contract — identical payloads re-encode to identical lengths. Re-derive the verdict from content on each use, or scope it to a single exclusive-lock hold.
- Long-lived supervision loops are allowed to heal transient failures, but only with bounded episodic budgets plus stable-uptime forgiveness. A process-lifetime restart cap that never resets is a latent outage mode: sparse isolated crashes can permanently disable auto-restart for the life of the daemon.
- Enum exhaustiveness checks across crates should treat cfg-gated variants as a contract boundary: declare and align feature gates in the consuming crate before using conditional match arms, rather than shipping broad fallbacks that silently hide missing feature paths.
- Live CDP harnesses should sanitize `TEMPO_CDP_CHROME` before spawn; shell-escaped paths (for example with `\ `) are common and must be unescaped to ` ` or tests can fail with opaque launcher errors ("No such file or directory (os error 2)"). Validate existence early so CI doesn't hang behind unclear I/O failures.
- CI filters that skip expensive security or engine build gates must be conservative. Changes to workflow files, lockfiles, manifests, gate scripts, the gated crate, and direct local dependency crates should run the gate unless a narrower proof is documented in the same change.
